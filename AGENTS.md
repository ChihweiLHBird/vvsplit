# AGENTS.md

## Fast Start

- `bunnysplit` is a static PyScript + MicroPython app. There is no backend, no app bundler, and no hand-written app JavaScript except `sw.js`.
- Start with executable sources of truth: `.github/workflows/ci.yml`, `.github/workflows/test.yml`, `.github/workflows/deploy.yml`, `tests/test_calc.py`, `scripts/stage-assets.sh`, `pyscript.toml`, and `infra/__main__.py`.
- `CLAUDE.md` is a symlink to this file; edit `AGENTS.md` and do not replace the symlink with an independent copy.
- Files under `docs/superpowers/specs/` and dated review reports are historical snapshots. Use current code, tests, and workflows when they disagree.

## Instruction Maintenance

- Repository behavior is the source of truth. If an agent finds that an `AGENTS.md` statement contradicts the code, tests, workflows, or deploy configuration, patch the nearest applicable `AGENTS.md` in the same change instead of preserving stale guidance.
- Every `AGENTS.md` in this repo has a matching `CLAUDE.md` symlink; preserve those links when changing instructions.

## Boundaries That Matter

- `main.py` only boots from persisted state: `ui.start(storage.load(), storage)`.
- Keep browser-only code in `ui.py` and `storage.py`. `splitcore/` must stay importable under plain CPython and under PyScript's MicroPython runtime.
- `pyscript.toml` is the import allowlist for MicroPython files copied into the virtual FS. If you add a runtime-imported Python module, update `pyscript.toml` or the browser build will fail even if local CPython tests pass.
- `dist/` is generated by `./scripts/stage-assets.sh`. Do not hand-edit `dist/`; edit source files and restage.
- `scripts/stage-assets.sh` currently copies the working tree through a denylist, not the Git index. `.gitignore` does not prevent a local file from reaching `dist/`; inspect staged output and keep local environments/tool state out until staging uses an explicit allowlist.
- Deploys are Pulumi-only from `infra/`; `pulumi up` uploads `../dist` to a Cloudflare assets-only Worker. There is no wrangler flow.
- CI stamps `dist/sw.js` with the commit SHA after staging; a local `stage-assets.sh` run does not. Before a manual `pulumi up`, ensure the staged `CACHE_VERSION` changes whenever app-shell content changes or installed clients can remain on stale cache entries.

## Repo-Specific Conventions

- Money is integer cents everywhere. Do not store amounts as floats or introduce float rounding into settlement logic.
- Keep `splitcore/model.py` on plain classes, not dataclasses, to preserve CPython/MicroPython behavior.
- In `ui.py`, build user-visible DOM with `createElement` plus `textContent`; avoid `innerHTML` for any user-controlled content.
- UI rerenders create JS proxies via `create_proxy`; if you add render-time handlers, keep them tracked in `_render_proxies` so rerenders do not leak proxies.
- `storage.load()` treats `localStorage` errors and corrupt JSON as recoverable and preserves bad payloads under `bunnysplit:corrupt`; keep that behavior instead of failing startup.
- `sw.js` is intentionally cache plumbing only. Do not move app logic into it.

## Validation

- App or core logic: `python -m unittest discover -s tests`
- Syntax parity with CI: `python -m py_compile main.py ui.py storage.py splitcore/*.py` and `python -m py_compile infra/__main__.py`
- Service-worker and staging syntax: `node --check sw.js` and `bash -n scripts/stage-assets.sh`
- Manual browser check: `python3 -m http.server`
- Before any deploy or deploy-related verification: `./scripts/stage-assets.sh`

## Hotspots

- Before changing `splitcore/calc.py` or `splitcore/model.py`: persisted-state hardening must never lose pennies or crash on malformed/oversized saved data. Amounts and uneven weights are capped at the model/calc boundary (`MAX_CENTS`, `_MAX_WEIGHT`); collection shapes and unique, bounded person/item IDs are part of the same trust boundary.
- `parse_cents()` must reject excessive digit counts before calling `int()`; checking `MAX_CENTS` afterward is too late to bound conversion cost or exceptions.
- Before changing `index.html` or `sw.js`: offline cold boot depends on the PyScript runtime being cached as a CORS-usable response, not opaque. Keep runtime warmup in default (cors) fetch mode, keep required app-shell installation atomic, and keep cache writes inside the fetch event lifetime.
- Save-status UI (`index.html`, `styles.css`, `ui.py`): the save-failure warning must never be color-only and must surface on mobile and at startup (`start()` probes `storage.writable()` and the `.err` label stays visible on narrow screens).
- Privileged workflow jobs contain cloud/state secrets. External `uses:` references should be verified full commit SHAs; major-version tags are not immutable.
