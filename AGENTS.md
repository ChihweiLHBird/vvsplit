# AGENTS.md

## Repo Focus

`vvsplit` is a static PyScript + MicroPython bill-splitting app. There is no
backend, no hand-written app JavaScript, and no separate app build step.

Start with these docs instead of duplicating them in changes:

- [Design spec](docs/superpowers/specs/2026-05-19-vvsplit-design.md)
- [Recent code review](docs/code-review-2026-05-24.md)
- [Infra and deploy notes](infra/README.md)

The design spec is useful background, but the live validation commands come
from [tests/test_calc.py](tests/test_calc.py) and
[.github/workflows/deploy.yml](.github/workflows/deploy.yml).

## Code Boundaries

- `main.py` boots the app from persisted state.
- `ui.py` is the DOM and event layer. Keep browser-specific code here.
- `storage.py` is the `localStorage` boundary.
- `splitcore/` must stay importable under plain CPython and MicroPython.
- `infra/` is Pulumi for the Cloudflare deployment. `pulumi up` is the deploy;
  there is no wrangler flow.

## Project Conventions

- Money is integer cents everywhere. Do not introduce float-based amount
  storage or float rounding into split calculations.
- Keep `splitcore/model.py` on plain classes, not dataclasses, so behavior
  stays consistent across CPython and MicroPython.
- In `ui.py`, build user-visible DOM with `createElement` + `textContent`.
  Avoid `innerHTML` for user-controlled content.
- If you add UI event handlers, keep proxy lifecycle aligned with
  `_render_proxies` cleanup so rerenders do not leak JS proxies.
- Treat `localStorage` failures and corrupt saved state as recoverable. Keep
  the current pattern of warning to the console and preserving bad payloads in
  `vvsplit:corrupt` instead of crashing startup.
- `dist/` is generated by [scripts/stage-assets.sh](scripts/stage-assets.sh).
  Edit source files in the repo root and restage assets instead of hand-editing
  generated files under `dist/`.

## Validation

Run the narrowest relevant checks after edits:

- Python app or core logic: `python -m unittest discover -s tests`
- Python syntax: `python -m py_compile main.py ui.py storage.py splitcore/*.py`
- Infra changes: `python -m py_compile infra/__main__.py`
- `index.html` or `sw.js` changes: run the JS-free audit from
  [.github/workflows/deploy.yml](.github/workflows/deploy.yml)
- Manual browser verification: `python3 -m http.server`

## Hotspots

- Uneven split changes and deserialization hardening must preserve exact penny
  conservation and never crash on malformed saved data. Read
  [docs/issues/01-uneven-split-large-finite-values.md](docs/issues/01-uneven-split-large-finite-values.md)
  before changing `splitcore/calc.py` or `splitcore/model.py`.
- Runtime loading, service worker caching, and startup saved-status behavior
  have known edge cases. Read the issue docs under [docs/issues](docs/issues)
  before changing `index.html`, `sw.js`, `main.py`, or startup UI flows.
- If a change affects save warnings or status presentation, account for the
  mobile accessibility issue documented in
  [docs/issues/05-save-failure-warning-is-color-only-on-mobile.md](docs/issues/05-save-failure-warning-is-color-only-on-mobile.md).
